## Auto-Delegation Patch

Problem found:

- Prompt-only rules in `SOUL.md` were not enough.
- A casual review request like "cek docs repo Naelvi_Studio, review aja ...
  jangan edit" was answered directly by Atlas using `cx/gpt-5.5`.

Fix added:

- File: `/opt/hermes-code/hermes-agent/gateway/run.py`
- A coding-instance gateway preprocessor detects review-only repository/docs
  requests and rewrites the message with a `ROUTING OVERRIDE`.
- The override requires first tool call to be `delegate_task`.
- It assigns exactly one Reviewer task using:
  - model: `cmc/deepseek/deepseek-v4-pro`
  - provider: `custom`
  - base_url: `http://127.0.0.1:20128/v1`
  - toolsets: `["file", "terminal"]`
  - read-only behavior.

Detection terms include review intent, repo/docs scope, and read-only wording:

- Review intent: `review`, `cek`, `audit`, `inspect`, `validate`, `verify`,
  `aman`, `aneh`, `kurang rapi`
- Scope: `repo`, `docs`, `Naelvi_Studio`, `repository`
- Read-only: `jangan edit`, `jangan ubah`, `review aja`, `cek doang`,
  `read-only`, `readonly`, `tanpa edit`

Backup file before patch:

- `/opt/hermes-code/hermes-agent/gateway/run.py.bak.review-route.*`

Team-mode style implementation/debug routing added:

- File: `/opt/hermes-code/hermes-agent/gateway/run.py`
- Implementation/debug prompts now get a `ROUTING OVERRIDE` that tells Atlas to
  coordinate an OmO-style multi-agent flow with `delegate_task`.
- The intended sequence is:
  1. Architect: read-only repo inspection, plan, risks, verification steps.
  2. Coder: implement the smallest safe change and run focused checks.
  3. Reviewer: read-only final diff/risk/test review.
- Model routing:
  - Architect: `ag/gemini-3.1-pro-low`
  - Coder: `cx/gpt-5.5`
  - Reviewer: `cmc/deepseek/deepseek-v4-pro`
  - Provider: `custom`
  - Base URL: `http://127.0.0.1:20128/v1`
- Review-only routing still runs first and remains read-only.
- Log marker: `code team routing override applied for inbound message`
- Backup file before patch:
  `/opt/hermes-code/hermes-agent/gateway/run.py.bak.team-route.`
- Follow-up guard added after smoke test:
  - Ambiguous requests like "benerin bug kecil" now trigger a clarification
    override instead of delegation.
  - Log marker: `code team routing clarification applied for inbound message`
  - Team route now explicitly says not to ask subagents to find arbitrary bugs.

## Telegram Delegation Progress UX

- When Atlas delegated to Architect/Coder/Reviewer, Telegram could feel silent
  for a long time because the gateway progress callback only displayed normal
  `tool.started` events.

  `tools/delegate_tool.py`, but filtered out by the Telegram gateway progress
  handler.

- File: `/opt/hermes-code/hermes-agent/gateway/run.py`

  - `Agent 1/3 mulai: ...`

  - `Agent 1/3 selesai (12s)` or `Agent 1/3 gagal: ...`

  Telegram view is informative but not too noisy.
